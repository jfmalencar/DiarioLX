\subsection{Relationship Overview} \label{sec:dm_relationships}

The main entities and their relationships are described below.

\subsubsection{User}
Represents an authenticated system user. A user has a role (\textit{Admin}, \textit{Editor}, or \textit{Contributor}) and a function (\textit{Journalist} or \textit{Photographer}). A user may author multiple content items; a content item may have multiple authors.

\subsubsection{Content}
The central entity of the system. A content item has a type (Article, Podcast, Video, or Photo Essay), a publication state, and references to its authors, categories, and tags. The body of a text article is stored as structured rich text, supporting inline images, subtitles, and highlights.

\subsubsection{Category and Subcategory}
Content is classified into a two-level category hierarchy. Each category has a name and an associated colour. A content item belongs to exactly one category (and optionally one subcategory).

\subsubsection{Tag}
Tags provide cross-cutting thematic classification. A content item may be associated with multiple tags; each tag has a dedicated listing page on the public site.

\subsubsection{Comment}
Readers (if registered) may submit comments on article pages. Comments are moderated by Editors and Administrators.

The physical data model maps the conceptual ER diagram to a relational schema in PostgreSQL. Table~\ref{tab:db_tables} lists the main database tables and their purpose.

\subsection{Key Design Decisions} \label{sec:dm_decisions}

\begin{itemize}
  \item \textbf{Unified content table}: All content types (articles, podcasts, videos, photo essays) share a common \texttt{content} table with a \texttt{type} discriminator column. Type-specific attributes are stored in separate extension tables.
  \item \textbf{State machine in the database}: The publication state of a content item is stored as an enumerated column in the \texttt{content} table, enforcing valid transitions at the application layer.
  \item \textbf{Media metadata separation}: Binary media files are not stored in the database; only their metadata (URL, size, MIME type) is persisted in the \texttt{media} table, with the actual files residing in the dedicated media storage system.
\end{itemize}
